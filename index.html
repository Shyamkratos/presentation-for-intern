Intern Projects — Android Security

This presentation showcases three major intern tasks: a Smali control-flow toolchain, an automated asset encryption & injection framework, and a research report on APK/AAB anti-decompilation techniques. All details from the original documentation are presented below.

Tasks

Task 1 — Control Flow Obfuscation Toolchain

Implements a Smali parser and CFG generator, restructures methods, injects opaque predicates, and recompiles APKs.

  • Modules: Engine.py, SmaliInfoExtractor.py, restructure_cfg.py, obfuscator.py, Assembler.py, packer.py, run.py
  • Flow: Decompile APK → Parse Smali → Build CFG → Restructure by method → Inject opaque predicates → Assemble → Repackage
  • Result: Harder static analysis due to obfuscated control flow.

Task 2 — AutoCrypInject Framework

Automates asset encryption using AES-256, generates Java decryption modules, converts to Smali, and patches APKs to decrypt at runtime.

  • Encryption: AES-256 CBC with PKCS7 padding, unique IV per file.
  • Injection: Java → class → dex → smali, then patch AssetManager.open()/openFd().
  • Flow: Decompile → Encrypt assets → Generate registry → Convert & inject → Rebuild & sign.

Task 3 — APK/AAB Anti-Decompilation Research

Explores methods to break reverse engineering tools like JADX and BundleTool while preserving APK functionality.

  • Success: Local File Header corruption breaks JADX parsing for APKs.
  • Failure: AAB locking unsuccessful due to BundleTool validation.
  • Conclusion: Comprehensive static anti-decompilation is not feasible; runtime protection is recommended.
